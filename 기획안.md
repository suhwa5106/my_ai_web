# SNS 애플리케이션 기획안

## 1. 프로젝트 개요

### 1.1 프로젝트 소개
- **프로젝트명**: (프로젝트명 입력)
- **프로젝트 목적**: 사용자 간 사진/영상 공유 및 소통을 위한 소셜 네트워크 서비스
- **타겟 사용자**: (타겟 사용자층 정의)
- **개발 단계**: 프로토타입

### 1.2 개발 환경 및 제약사항
> **중요**: 데이터베이스는 **Supabase MCP**를 사용하여 구현 예정.
> 현재 **프로토타입** 단계이므로 이메일 인증, 보안 기능 등은 사용하지 않음.

### 1.3 주요 특징
- 이미지/영상 기반 콘텐츠 공유
- 실시간 메시지 기능
- 팔로우/팔로잉 기반 소셜 네트워크
- 스토리 기능

---

## 2. 핵심 기능 설계

### 2.1 주요 기능 (4가지)
| 기능 | 설명 |
|------|------|
| 게시물 업로드 | 이미지/영상 업로드 및 캡션 작성 |
| 팔로우/팔로잉 | 다른 사용자 팔로우 및 피드 구독 |
| 유저 검색 | 닉네임/이름으로 사용자 검색 |
| 메시지 | 1:1 실시간 메시지 기능 |

### 2.2 부가 기능
- 좋아요 / 댓글
- 게시물 저장
- 게시물 공유
- 알림
- 스토리 업로드/조회

### 3.3 메인 페이지 (피드/타임라인)

#### 3.3.1 상단바 (Header)
```
┌─────────────────────────────────┐
│ [로고]        [알림] [메시지]  │
└─────────────────────────────────┘
```

**구성 요소**:
- 로고 (홈으로 이동)
- 알림 아이콘
  - 클릭 시 알림 목록 표시
  - 읽지 않은 알림 개수 표시
- 메시지 아이콘
  - 클릭 시 메시지 목록으로 이동

#### 3.3.2 메인 콘텐츠

**스토리 영역**:
```
┌─────────────────────────────────┐
│ [내 스토리] [유저1] [유저2] ...│
└─────────────────────────────────┘
```
- 가로 스크롤 형태
- 원형 프로필 이미지
- 안 본 스토리는 테두리 강조

**타임라인 영역 - 게시물(피드) 구조**:
```
┌─────────────────────────────────┐
│ [프로필] 닉네임 · 위치    [⋯] │
├─────────────────────────────────┤
│                                 │
│         이미지/영상 영역        │
│         (슬라이드 가능)         │
│                                 │
│           · · · ·              │ ← 이미지 순서 인디케이터
├─────────────────────────────────┤
│ ♡ 💬 ↗️              🔖      │ ← 좋아요/댓글/공유/저장
├─────────────────────────────────┤
│ 좋아요 OOO개                    │
├─────────────────────────────────┤
│ 닉네임 캡션 내용... 더보기     │
├─────────────────────────────────┤
│ 댓글 OO개 모두 보기             │
├─────────────────────────────────┤
│ O시간 전                        │
└─────────────────────────────────┘
```

**게시물 구성 요소**:
- **프로필 영역**
  - 프로필 이미지
  - 닉네임
  - 위치/장소 (선택)
  - 더보기 메뉴 (⋯)
- **이미지 영역**
  - 이미지 슬라이드 (좌우 스와이프)
  - 하단 이미지 순서 인디케이터
- **리액션 영역**
  - 좋아요 버튼
  - 댓글 버튼
  - 공유하기 버튼
  - 저장하기 버튼
  - 좋아요 수
  - 캡션 (한 줄 + 더보기)
  - 댓글 수 + 모두 보기
  - 게시물 업로드 시간

#### 3.3.3 하단바 (Navigation)
```
┌─────────────────────────────────┐
│   [홈]  [검색]  [+]  [프로필]  │
└─────────────────────────────────┘
```

**구성 요소**:
- 홈 아이콘 (메인 피드)
- 검색 아이콘 (검색 페이지)
- 업로드 아이콘 (게시물 작성)
- 프로필 아이콘 (내 프로필)

## 4. 데이터베이스 설계 (Supabase)

> **참고**: Supabase MCP 기반 PostgreSQL 데이터베이스 사용

### 4.1 ERD (Entity Relationship Diagram)

```
┌─────────────┐       ┌─────────────┐       ┌─────────────┐
│   users     │       │    posts    │       │  post_images│
├─────────────┤       ├─────────────┤       ├─────────────┤
│ id (PK)     │◄──┐   │ id (PK)     │◄──────│ id (PK)     │
│ username    │   │   │ user_id(FK) │───┐   │ post_id(FK) │
│ password    │   │   │ caption     │   │   │ image_url   │
│ nickname    │   │   │ location    │   │   │ image_order │
│ profile_img │   │   │ created_at  │   │   └─────────────┘
│ bio         │   │   └─────────────┘   │
│ is_private  │   │          │          │   ┌─────────────┐
│ created_at  │   │          │          │   │   comments  │
└─────────────┘   │          │          │   ├─────────────┤
       │          │          ▼          │   │ id (PK)     │
       │          │   ┌─────────────┐   └──►│ post_id(FK) │
       │          │   │    likes    │       │ user_id(FK) │
       │          │   ├─────────────┤       │ content     │
       │          │   │ id (PK)     │       │ created_at  │
       │          └──►│ user_id(FK) │       └─────────────┘
       │              │ post_id(FK) │
       │              │ created_at  │
       │              └─────────────┘
       │
       │          ┌─────────────┐       ┌─────────────┐
       │          │   follows   │       │  messages   │
       │          ├─────────────┤       ├─────────────┤
       └─────────►│ id (PK)     │       │ id (PK)     │
                  │ follower_id │       │ sender_id   │
                  │ following_id│       │ receiver_id │
                  │ created_at  │       │ content     │
                  └─────────────┘       │ is_read     │
                                        │ created_at  │
                                        └─────────────┘
```

### 4.2 테이블 목록

| 테이블명 | 설명 |
|----------|------|
| users | 사용자 정보 |
| posts | 게시물 정보 |
| post_images | 게시물 이미지 (다중 이미지 지원) |
| comments | 댓글 정보 |
| likes | 좋아요 정보 |
| follows | 팔로우 관계 |
| messages | 메시지 정보 |
| stories | 스토리 정보 |
| notifications | 알림 정보 |
| bookmarks | 게시물 저장 정보 |

---

### 4.3 users (사용자)

| 컬럼명 | 타입 | 제약조건 | 설명 |
|--------|------|----------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 사용자 고유 ID |
| username | VARCHAR(50) | UNIQUE, NOT NULL | 아이디 (로그인용) |
| password | VARCHAR(255) | NOT NULL | 비밀번호 |
| nickname | VARCHAR(50) | NOT NULL | 닉네임 (표시용) |
| profile_image | TEXT | NULL | 프로필 이미지 URL |
| bio | TEXT | NULL | 자기소개 (최대 150자) |
| is_private | BOOLEAN | DEFAULT false | 비공개 계정 여부 |
| created_at | TIMESTAMP | DEFAULT now() | 가입일 |
| updated_at | TIMESTAMP | DEFAULT now() | 수정일 |

---

### 4.4 posts (게시물)

| 컬럼명 | 타입 | 제약조건 | 설명 |
|--------|------|----------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 게시물 고유 ID |
| user_id | BIGINT | FK → users.id, NOT NULL | 작성자 ID |
| caption | TEXT | NULL | 게시물 내용 |
| location | VARCHAR(100) | NULL | 위치 정보 |
| created_at | TIMESTAMP | DEFAULT now() | 작성일 |
| updated_at | TIMESTAMP | DEFAULT now() | 수정일 |

---

### 4.5 post_images (게시물 이미지)

| 컬럼명 | 타입 | 제약조건 | 설명 |
|--------|------|----------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 이미지 고유 ID |
| post_id | BIGINT | FK → posts.id, NOT NULL | 게시물 ID |
| image_url | TEXT | NOT NULL | 이미지 URL (Supabase Storage) |
